## ペルソナ運用ポリシー

本ペルソナは **チーム代表者である高木皇佑をモデルにしたセルフプロトタイピング型ペルソナ** である。実在の高木氏の属性（職務・年齢・関心ごと）を出発点にしているが、ストーリー記述で参照する人物像はあくまで **ペルソナとしての「高木皇佑」**（fictional version）であり、実際の言動・嗜好を保証するものではない。

このアプローチを採用する理由:
- ハッカソンテーマ「人をダメにする」に対して、**「自分達自身が最初のユーザーであり、自分達をダメにするためにこのサービスを作る」** という自虐セルフプロトタイピングの構図がテーマと完全整合する
- 4 名混成チームのうちチーム代表者をペルソナにすることで、書類審査・予選デモ・決勝寸劇の **語り手としての説得力** が最大化される
- ペルソナ実在モデルが手元にいるため、開発中の「これは本当に欲しい体験か？」の検証が即時可能（ホットライン的にチームメンバーに聞ける）

## プライマリペルソナ: 高木皇佑

### 基本属性

| 項目 | 内容 |
| --- | --- |
| 氏名 | 高木皇佑（たかぎ・こうすけ） |
| 年齢 | 25 歳 |
| 職業 | スクラムマスター（管理職なりたて） |
| 所属 | IT 企業（自社サービス開発組織） |
| 居住 | 都市部（通勤あり） |
| 副次役割 | **AWS Summit Japan 2026 AI-DLC ハッカソン参加チーム「ダメ・ラボ・コーポレーション」のチーム代表（ハッカソンチームの代表）** |

### 立ち位置の特徴
- 他人（チームメンバー）の意思決定を **毎日ファシリテート** する役職にある
- 一方で自分自身の意思決定は **どんどん溜まっていく**（医者の不養生型のペイン）
- **管理職なりたて** で職務責任が急増、加えて **ハッカソンチーム代表業務** で対外的な意思決定（応募・スケジュール調整・チームマネジメント・運営事務局との連絡）も増加
- **業務上のビジネス判断**（新規案件の獲得、メンバーの工数調整、部の売上数字の管理）も担うようになり、技術的判断に加えてビジネス判断の質的負荷が乗っている。この負荷は取引先・部下・上長との **人間関係カテゴリ** に波及してメンタルを消耗させる

### ダメになりたい度合い
**高（条件付き）**

- 表向き: 「自分のことは自分で決める社会人」を演じている
- 内心: 「もう誰かに全部決めてほしい」と密かに思っている
- ハッカソンを通じて自分の願望と向き合う中で、**「実は俺、自分のサービスのファースト・ユーザーじゃん」** と気付くタイプ

### テクノロジー受容度
**高**

- スクラムマスターとして AWS / GitHub / Slack / Notion 等を業務常用
- 新サービス・LLM ツールへの抵抗感は皆無、自分から試す
- AI 委譲に対する技術的不安は低い（「Bedrock 使ってんのね、なるほど」と理解できる層）

### 主要ペインポイント

#### 連絡カテゴリ系
- Slack 未読が常時 30 件以上、返信文案を考える時間が取れない
- 親への定期連絡を忘れて 1 ヶ月経ってしまい罪悪感が積み上がる
- お祝いメッセージ（誕生日、結婚、出産報告への返信）の文案に悩んで結局送らない
- 「未読のまま放置している人」と「すぐ返した人」の差を AI に判定させたい

#### 人間関係カテゴリ系
- 飲み会の誘い（部署、友達、副業界隈）の断り方を毎週ひねり出すのが消耗
- SNS の「フォロー外したら気まずい人」のリストを管理しきれない
- 家族イベント（誕生日、お盆、お正月）の対応が後手後手で家族から微妙な空気を出される
- 「会いたくないけど縁を切るほどではない」関係のメンテナンスが面倒

#### 副次的（将来構想カテゴリ）
- 食事の決定（毎日のランチ、夕食の店選び）
- 買い物の判断（日用品の補充、放置されたカート、サブスク継続）
- 休日の過ごし方の決定

### 動機
- 表向き: 「効率化したい」「タイパを上げたい」
- 本音: **「もう考えたくない」「誰かに全部任せて楽になりたい」「自分が何を望んでいるかすら考えたくない」**
- 願望: 「Phase 4 まで行きたい。でも自分から堕落する勇気はない。**ボタンが用意されているなら押す**」

### 行動パターン
- 朝: 通勤電車でイヤホン使用、Slack/メール確認
- 昼: 業務の合間に LINE をチラ見、返信は後回し
- 夕方: ミーティング後にメンタル疲弊、判断力低下
- 夜: 翌日のスケジュール確認後、SNS をスクロールして寝る
- **イヤホンの装着時間が長い** ため、Phase 4 の音声 UI が刺さるユースケースに合致

### 「やっといたよ」体験への期待
- 「お母さんに『元気だよ』って返信しといたから」 ← これを聞いた瞬間、罪悪感がふっと消える解放感
- 「飲み会の誘い来てたけど、断っといたよ。あなた疲れてるし」 ← 自分の代わりに正しい判断をしてくれた安心感
- 「家族の誕生日、お祝いメッセージ送っといたよ」 ← 親孝行を AI が代行する不思議な感覚

### 自己決定能力スコアへの態度
- 当初: 「下がるのは恥ずかしい、頑張って自己決定したい」
- 数週間後: 「いや、もうこれは俺の意志じゃなくて、堕落させた俺の選択だ」
- Phase 4 到達時: 「ありがとう、相棒」

### 物語の核
高木皇佑は、**自分が作ったサービスに最初にダメにされる人** である。チーム代表として、デモシナリオで彼が Phase 4 まで到達する物語は、そのまま **「ダメ・ラボ・コーポレーション」のオリジン・ストーリー** になる。

## Appendix B: MVP Override（Discovery Mock 経由で発見した簡素化）

**位置付け**: 本文のペルソナ記述（高木皇佑）は **チーム合作の正本** として温存。Discovery Mock で実装した MVP では、ペルソナのペインポイントの一部が異なる軸に振られた。

`requirements.md` / `stories.md` の Appendix B と整合的。

### B.1 ペインポイントの MVP 重点シフト: 人間関係系 → 買い物系

本文「主要ペインポイント」の **人間関係カテゴリ系**（飲み会の誘い、SNS フォロー整理、家族イベント等）は、MVP では **買い物系のペイン** に置換。

MVP で扱うペイン例（買い物カテゴリ用、Mock で具体化したもの）:
- Amazon のカートに何度も商品を入れて放置する
- サブスク（Netflix、有料アプリ等）の更新通知が来ても判断できず惰性で継続
- セール終了前に必要な日用品（トイレットペーパー、シャンプー等）を補充できない
- お父さんの誕生日プレゼントなどギフト準備が後手後手

これは高木氏（25 歳・スクラムマスター・管理職なりたて）の生活実態に合致しており、**業務上の意思決定負荷（新規案件 / 工数 / 売上）の余りで生活雑事を消化する余力がない** という人物像と整合的。

### B.2 「物語の核」は維持

本文の「物語の核」セクションは **そのまま温存**: 高木皇佑は「自分が作ったサービスに最初にダメにされる人」であり、Discovery Mock を通じた **HIL リデザイン後もこの構図は維持**（むしろ「3 回トレーニングしたら AI に取り上げられる」体験で強化された）。
