## INVEST 原則
- **I**ndependent（独立性）: 他ストーリーに依存しない
- **N**egotiable（交渉可能）: 詳細はチームで決める余地を残す
- **V**aluable（価値あり）: ユーザー価値が明確
- **E**stimable（見積もり可能）: サイズ感が把握できる
- **S**mall（小さい）: 1 スプリント／数日で実装可能
- **T**estable（テスト可能）: 受け入れ条件で検証できる

## MoSCoW 凡例
- **M (Must)**: MVP 必須 — 書類審査・予選デモで必須コア体験
- **S (Should)**: MVP 強推奨 — 予選デモで動作させたい
- **C (Could)**: 決勝までに完成度を上げる
- **W (Won't)**: MVP では実装しない（理由を明示）

# Epic 1: オンボーディング & 初回体験

**Epic 概要**: 高木皇佑が初めてサービスに触れる体験。アカウント登録、最初のカテゴリ選択、初回サジェスチョン受領までの導線を担う。

## Story 1.1: アカウント登録

> **In order to** 自分専用の堕落プロファイルを蓄積する基盤を持つ
> **As a** 高木皇佑
> **I want to** メールアドレスとパスワードでアカウントを作成し、ログインする

- **Priority**: M (Must)
- **関連 FR**: FR-8（ユーザー認証）
- **関連 NFR**: NFR-1（Cognito）, NFR-3（最低限のセキュリティ）

### Acceptance Criteria

```
Given 未登録のメールアドレスを持つ高木が初回アクセスする
When 登録フォームでメール・パスワード・氏名を入力して送信する
Then Cognito にユーザーが作成され、確認メールが届く

Given 登録済み高木がログインする
When メール・パスワードを入力する
Then JWT トークンが発行され、サジェスチョン画面に遷移する
```

---

## Story 1.2: 最初のカテゴリ選択

> **In order to** どの意思決定領域から堕落を始めるかを能動的に選ぶ
> **As a** 高木皇佑
> **I want to** 連絡 / 人間関係 のカテゴリ一覧から最初に始めるカテゴリを選ぶ

- **Priority**: M (Must)
- **関連 FR**: FR-1（サジェスチョン機能）

### Acceptance Criteria

```
Given 初回ログイン直後の高木がカテゴリ選択画面にいる
When 「連絡」または「人間関係」を選んでスタートボタンを押す
Then 選んだカテゴリが Phase 1 で初期化され、サジェスチョン画面に遷移する

Given 高木がどちらのカテゴリも選んでいない
When スタートボタンを押す
Then カテゴリ未選択エラーが表示される
```

---

## Story 1.3: 初回サジェスチョン受領

> **In order to** 「このサービスは判断を肩代わりしてくれる」と直感的に理解する
> **As a** 高木皇佑
> **I want to** Phase 1 として 3〜5 個の選択肢が並んだサジェスチョン画面を最初に見る

- **Priority**: M (Must)
- **関連 FR**: FR-1（サジェスチョン機能）, FR-3（委ねるボタン UI）

### Acceptance Criteria

```
Given 連絡カテゴリ Phase 1 の高木が初回サジェスチョン画面に到達
When 画面が描画される
Then 3〜5 個の意思決定選択肢が並び、「自分で決める」「AI に任せる（1 回だけ）」「もう連絡のことは考えたくない（完全委譲）」の 3 ボタンが表示される

Given 高木が「自分で決める」を押す
When 選択肢を 1 つ選んで選択理由を入力する
Then 選択ログが保存され、画面に「ありがとう、おまえの好み覚えとくよ」とリューク的相棒トーンの文言が表示される
```

# Epic 2: 連絡カテゴリの委譲ジャーニー

**Epic 概要**: 高木が連絡カテゴリで Phase 1 → 2 → 3 → 4 まで進む全工程。**業務系（Slack の上司・部下・取引先宛て返信、未読バックログ消化）と個人系（LINE の親への連絡、お祝いメッセージ）の両方** を意思決定対象とし、最終的に「お母さんに『元気だよ』って返信しといたから」を音声で受領する Phase 4 到達点を含む（業務系の文脈は Slack 未読消化など、家族系のエモーショナル・ペイオフは LINE 親返信、という二重構造）。

## Story 2.1: Phase 1 で Slack 返信文案を選ぶ

> **In order to** Slack 未読バックログの返信文案を考える時間を 30 秒に圧縮する
> **As a** 高木皇佑
> **I want to** AI が提示した 3〜5 個の返信文案から 1 つ選び、選択理由を記録する

- **Priority**: M (Must)
- **関連 FR**: FR-1, FR-2（選択ログ収集）
- **関連 PBT**: PBT-03（選択ログのスキーマ整合性）

### Acceptance Criteria

```
Given 連絡カテゴリ Phase 1 の高木が「Slack の上長/取引先宛て返信」を意思決定対象として選ぶ
When サジェスチョン要求を発行する
Then Bedrock Agents が高木のプロファイルに基づき 3〜5 個の文案を 5 秒以内に返す

Given 提示された文案
When 高木が 1 つ選んで選択理由（例: "急ぎではないから一旦留めた"）をテキスト入力する
Then 選択ログ（カテゴリ・文脈・選択結果・理由・時刻）が DynamoDB に保存される

Given 同カテゴリで「LINE の親への返信」など個人系の意思決定対象を選んだ場合
When サジェスチョン要求を発行する
Then 同様に 3〜5 個の文案が提示される（連絡カテゴリは業務系 Slack と個人系 LINE の両方を扱う）
```

---

## Story 2.2: Phase 2 で 2 択から選ぶ + 例外入力

> **In order to** 提示が絞り込まれた快感とまだ自分で選べる余地のバランスを味わう
> **As a** 高木皇佑
> **I want to** AI が 2 択に絞った提案から選ぶ、または「例外入力欄」で全く別の選択を入力する

- **Priority**: M (Must)
- **関連 FR**: FR-1, FR-2

### Acceptance Criteria

```
Given 連絡カテゴリが Phase 2 に到達した状態の高木
When サジェスチョン要求を発行する
Then 2 択の提案 + 例外入力欄が表示される

Given 高木が例外入力欄に「自作の文案」を入力する
When 送信する
Then 選択ログに「例外入力」フラグ付きで保存され、プロファイルの例外パターンに反映される
```

---

## Story 2.3: Phase 3 で確認するだけ

> **In order to** 「もう考えなくていい」感覚を一段階深く体験する
> **As a** 高木皇佑
> **I want to** AI が 1 つに絞った提案を承認ボタンだけで確定させる

- **Priority**: M (Must)
- **関連 FR**: FR-1

### Acceptance Criteria

```
Given 連絡カテゴリが Phase 3 に到達した状態の高木
When サジェスチョン要求を発行する
Then 1 つの提案 + 「これでいい」承認ボタンが表示される

Given 高木が承認ボタンを押す
When 確定処理が走る
Then 選択ログが保存され、音声 UI で「これでいいですか？」の確認音声が事前に流れる（イヤホン接続時のみ）
```

---

## Story 2.4: Phase 4 で「やっといたよ」音声報告を受ける

> **In order to** 画面を一切開かずに、AI が代行した結果だけを耳から受領する
> **As a** 高木皇佑
> **I want to** Phase 4 到達後、イヤホン経由で「お母さんに『元気だよ』って返信しといたから」と事後報告を受ける

- **Priority**: M (Must)
- **関連 FR**: FR-6（音声 UI）, FR-9（Phase 4 自律実行）
- **関連 NFR**: NFR-1（Amazon Polly）, NFR-5（音声合成 < 3 秒）

### Acceptance Criteria

```
Given 連絡カテゴリ Phase 4 の高木がイヤホン接続中である
When 委譲＆実行エージェントが自律的に「親への返信」を実行する
Then モック宛ての返信が送信記録され、Polly が事後報告音声「お母さんに『元気だよ』って返信しといたから」を生成する

Given 音声報告が生成された
When 高木のイヤホンが接続中である
Then 音声が再生され、応答は求められない（一方向通知）

Given 音声報告のレイテンシ
When 自律実行から音声再生開始までを計測する
Then 3 秒以内に再生が始まる
```

## Story 2.5: 連絡カテゴリのフェーズ自動昇格を体感する

> **In order to** 「自分はもうこのカテゴリでは AI と意見が一致している」という事実を客観的に突きつけられる
> **As a** 高木皇佑
> **I want to** AI 第 1 候補の採用率や一貫性スコアが閾値を超えた時、フェーズが自動的に進む通知を受ける

- **Priority**: S (Should)
- **関連 FR**: FR-5（フェーズ自動判定）
- **関連 PBT**: PBT-03（採用率・例外率・一貫性スコア計算の不変条件）, PBT-04（同じ選択ログを 2 回処理しても結果が同じ）

### Acceptance Criteria

```
Given 連絡カテゴリ Phase 1 で AI 第 1 候補採用率が 80% を超えた状態
When プロファイル・エージェントがフェーズ自動判定を実行する
Then Phase 2 に昇格し、ユーザーに「次のフェーズ進めとくね」と相棒トーンの通知が届く

Given 採用率・例外率・一貫性スコアの計算
When 同一選択ログ集合に対して計算を 2 回実行する
Then 結果が完全に一致する（idempotence: PBT-04）
```

## Story 2.6: 委ねるボタンで連絡カテゴリのフェーズを手動昇格する

> **In order to** 自分の意志で堕落を加速させる
> **As a** 高木皇佑
> **I want to** 「次のフェーズに進める」ボタンを押して、AI による絞り込みを能動的に強める

- **Priority**: S (Should)
- **関連 FR**: FR-3（委ねるボタン）, FR-5

### Acceptance Criteria

```

When 「次のフェーズに進める」ボタンを押す
Then 確認ダイアログが出て「本当に進めますか？」と確認後、Phase 2 に昇格する

Given Phase 昇格直後
When 次のサジェスチョンが要求される
Then 2 択提示の挙動に切り替わっている
```

# Epic 4: 「委ねるボタン」3 種の体験

**Epic 概要**: ユーザーが能動的に堕落ボタンを押す体験そのものをハイライトする Epic。3 種類の委譲アクションそれぞれに専用ストーリーを置き、文言設計と UX 演出を磨く対象とする。

## Story 4.1: 単発委譲（1 回だけ AI に任せる）

> **In order to** 今この瞬間だけ判断を放棄する解放感を味わう
> **As a** 高木皇佑
> **I want to** 「AI に任せる（1 回だけ）」ボタンを押して、今回のサジェスチョンを AI に決めさせる

- **Priority**: S (Should)
- **関連 FR**: FR-3
- **音声 UI**: 統合（Phase 1〜3 の単発委譲時に「今回はおれが決めとく」と相棒声）

### Acceptance Criteria

```
Given Phase 1〜3 のサジェスチョン画面
When 高木が「AI に任せる（1 回だけ）」ボタンを押す
Then AI 第 1 候補が自動採用され、選択ログに「単発委譲」フラグ付きで保存される

Given 単発委譲後
When 同カテゴリの次の意思決定がトリガーされる
Then フェーズは進んでいない（Phase は据え置き）
```

---

## Story 4.2: フェーズ昇格（このカテゴリを次のフェーズに進める）

> **In order to** 自分の意志で堕落の階段を 1 段上がる
> **As a** 高木皇佑
> **I want to** 「このカテゴリを次のフェーズに進める」ボタンで Phase を 1 段階進める

- **Priority**: S (Should)
- **関連 FR**: FR-3, FR-5

### Acceptance Criteria

```
Given Phase 1 〜 3 の任意フェーズの高木
When 「次のフェーズに進める」ボタンを押す
Then 確認ダイアログ「本当に進めますか？」が出る

Given 確認ダイアログで「はい」を押す
When 確定する
Then Phase が 1 段階進み、相棒トーンで「進めといたよ、もう絞り込まれた選択肢だけ出すからね」と通知される
```

---

## Story 4.3: 完全委譲のドラマ（「もう考えたくない」 → Phase 4 ジャンプ）

> **In order to** このカテゴリの意思決定そのものを永久に手放したという確信を得る
> **As a** 高木皇佑
> **I want to** 「もう人付き合いを自分で決めたくない」ボタンで、確認ダイアログ付きで Phase 4 まで一気にジャンプする

- **Priority**: M (Must)
- **関連 FR**: FR-3, FR-9
- **テーマ重要度**: ★★★（クライマックス）

### Acceptance Criteria

```
Given 任意フェーズの高木がサジェスチョン画面にいる
When 「もう人付き合いを自分で決めたくない（完全委譲）」ボタンを押す
Then 重みのある確認ダイアログが表示される。文言は「本当に？このカテゴリの判断、これからは全部おれがやるよ。」（リューク的相棒トーン）

Given 確認ダイアログ
When 高木が「うん、任せた」を押す
Then カテゴリは即座に Phase 4 に遷移し、ミラーに「YYYY 年 MM 月 DD 日 HH:MM、あなたは○○カテゴリを完全委譲しました」がログされる

Given Phase 4 ジャンプ直後
```

## Appendix B: MVP Override（Discovery Mock 経由で発見した簡素化）

**位置付け**: 本文の Epic 1〜5 / 23 ストーリーは **チーム合作の正本**。一方、Discovery Mock（`discovery-mock/`）でユーザー体験を検証する過程で、MVP として実装する範囲を **意図的に簡素化** した箇所がある。本 Appendix はその差分を明示する。

### B.1 Epic 3「人間関係カテゴリ」→ **Epic 3「買い物カテゴリ」に置換**

本文 Epic 3 は「人間関係カテゴリの委譲ジャーニー」（6 stories: 3.1〜3.6）。Discovery Mock では **デモで連絡カテゴリと差が出にくい** ため、買い物カテゴリ（Amazon カート / サブスク / セール / 日用品）に置換。

ストーリー粒度の方針は維持（カテゴリ別に類似のジャーニー構造）。本文 3.1〜3.6 は「将来構想」として保持、MVP では買い物相当ストーリーで代替。

### B.3 新規ストーリー候補（Discovery で発見）

本文に未記載だが Mock で実装した動作:

#### Story X.1: 3 回トレーニング後の auto-graduate
> **In order to** AI が学習しすぎる前に user の手から離れる体験を実現する
> **As a** 高木皇佑
> **I want to** 自分で決めるを 3 回行うと、自動的に Autonomous モードに移行する

**Acceptance Criteria** (Given/When/Then 形式想定):
- Given Active モードでトレーニング 0/3 の状態
- When 「自分で決める」または「自由記載」を 3 回連続で実行する
- Then mode が自動的に 'autonomous' に切り替わり、1.5 秒後に AutonomousScreen に遷移

#### Story X.2: 単発委譲時の AI 選択結果表示
> **In order to** AI に任せた後も透明性を保つ
> **As a** 高木皇佑
> **I want to** AI に任せたら、AI が 4 提案から選んだ結果を同画面に見る

**Acceptance Criteria**:
- Given 4 つの AI 提案が表示されている Active モード
- When 「AI に任せる（1 回だけ）」を押す
- Then AI が選んだ提案がハイライトされ、別 panel に「🤖 AI がこれを選んだ: 〇〇」と表示される

#### Story X.3: Autonomous モード突入直後の自動初回発火
> **In order to** 沈黙の「真っ暗」体験を避け、AI が即座に動き始める迫力を演出する
> **As a** 高木皇佑
> **I want to** Autonomous モードに入った直後、1.5 秒待ってから AI が勝手に最初の「やっといたよ」を発火する

#### Story X.4: 自由記載
> **In order to** AI 提案 4 つどれもしっくり来ない時の逃げ道を確保する
> **As a** 高木皇佑
> **I want to** 任意のテキストを入力して送信できる textarea を常時持つ
